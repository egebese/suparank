## What is the Model Context Protocol (MCP)?

The [Model Context Protocol](https://modelcontextprotocol.io/) is an open standard introduced by Anthropic in November 2024 to standardize how AI assistants connect to external tools and data sources. Think of it as a universal adapter that lets your AI assistant talk to databases, APIs, content management systems, and specialized tools—without custom integrations for each one. For a complete introduction to the protocol, see our guide on [what is Model Context Protocol](/blog/what-is-model-context-protocol).

<Callout type="tip" title="Why MCP Matters for Content Writers">
Before MCP, you'd ask Claude for a blog post, copy it to WordPress, run SEO analysis separately, find images on another platform, then manually optimize and publish. With MCP, your AI assistant can research keywords, write content, generate images, optimize for SEO, and publish to WordPress—all in one conversation.
</Callout>

### How MCP Works: The Technical Architecture

MCP uses a client-server architecture where:

1. **MCP Client**: Your AI assistant (Claude Desktop, Cursor, ChatGPT)
2. **MCP Server**: The tool you're connecting to (Suparank, Notion, GitHub, etc.)
3. **Transport**: How they communicate (stdio for local, SSE for remote)

```
┌─────────────────┐         ┌─────────────────┐         ┌─────────────────┐
│  Your AI Client │ ◄─────► │   MCP Server    │ ◄─────► │  External APIs  │
│ (Claude/Cursor) │  stdio  │  (Suparank)     │  HTTPS  │ (WordPress/etc) │
└─────────────────┘         └─────────────────┘         └─────────────────┘
```

When you ask your AI assistant to "research keywords about coffee brewing," the AI:
1. Identifies it needs the `keyword_research` tool from your MCP server
2. Calls the tool through MCP protocol
3. The MCP server executes the tool (queries APIs, processes data)
4. Returns structured results back to the AI
5. AI presents results in conversational format

<Callout type="info">
**Industry Adoption**: As of January 2026, MCP is supported by Claude Desktop, Cursor, ChatGPT (since March 2025), and multiple open-source AI clients. Major companies like OpenAI, Block, and Anthropic co-founded the Agentic AI Foundation under the Linux Foundation to govern MCP's development.
</Callout>

## Step-by-Step: Setting Up MCP with Claude Desktop

Let's start with the most popular setup: Claude Desktop with an MCP content writing tool.

### Step 1: Install Claude Desktop

If you haven't already:

1. Go to [claude.ai/download](https://claude.ai/download)
2. Download for your operating system
3. Install and sign in with your Anthropic account

### Step 2: Locate Configuration File

Claude Desktop uses a JSON configuration file to register MCP servers. The file location depends on your operating system:

**macOS:**
```
~/Library/Application Support/Claude/claude_desktop_config.json
```

**Windows:**
```
%APPDATA%\Claude\claude_desktop_config.json
```

**Quick way to open it:**

**macOS:**
```bash
open ~/Library/Application\ Support/Claude/claude_desktop_config.json
```

**Windows (PowerShell):**
```powershell
notepad $env:APPDATA\Claude\claude_desktop_config.json
```

If the file doesn't exist, create it with empty JSON: `{}`

### Step 3: Add MCP Server Configuration

Now we'll add an MCP server to the configuration. Let's use three examples: a simple memory server, the filesystem server, and Suparank for full blog automation.

**Example 1: Memory Server (Simple Test)**

```json
{
  "mcpServers": {
    "memory": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-memory"]
    }
  }
}
```

This adds a simple key-value storage server to test MCP functionality.

**Example 2: Filesystem Server (Local File Access)**

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/yourname/Documents/blog-content"
      ]
    }
  }
}
```

Replace `/Users/yourname/Documents/blog-content` with your actual content folder path.

**Example 3: Suparank (Full Blog Automation)**

```json
{
  "mcpServers": {
    "suparank": {
      "command": "npx",
      "args": ["suparank", "your-project-slug", "your-api-key"]
    }
  }
}
```

Replace `your-project-slug` and `your-api-key` with your actual Suparank credentials (get them at [app.suparank.io](https://app.suparank.io/)).

**Multiple Servers (Recommended Setup):**

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/yourname/Documents/blog-content"
      ]
    },
    "suparank": {
      "command": "npx",
      "args": ["suparank", "my-blog", "sk_live_abcd1234..."]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_your_token_here"
      }
    }
  }
}
```

<Callout type="tip" title="Pro Tip: Organizing Multiple Servers">
You can add as many MCP servers as you want. Common combinations for content teams:
- **Filesystem** (local file access)
- **Suparank** (SEO + content automation)
- **GitHub** (version control for content)
- **Notion** (documentation and planning)
- **Slack** (team notifications)

The AI will intelligently choose which tool to use based on your request.
</Callout>

### Step 4: Restart Claude Desktop

After saving the configuration file:

1. **Completely quit** Claude Desktop (don't just close the window)
   - macOS: `Cmd + Q`
   - Windows: Right-click taskbar icon → Quit
2. **Relaunch** Claude Desktop

### Step 5: Verify MCP Connection

Look for the MCP indicator in Claude Desktop:

1. Start a new conversation
2. Look at the bottom-right of the input box
3. You should see a small icon (looks like a puzzle piece or tool icon)
4. Click it to see available MCP tools

**Test the connection:**

Type in Claude:
```
What MCP tools do I have available?
```

Claude should list your configured servers and their tools.

**Test a tool:**

```
Use the memory server to save this: "My blog is about specialty coffee"
```

If you get a confirmation, MCP is working perfectly.

<Callout type="warning">
**Troubleshooting**: If you don't see the MCP indicator after restarting:
1. Check JSON syntax (use [jsonlint.com](https://jsonlint.com/) to validate)
2. Verify Node.js is installed: `node --version` in terminal
3. Check file path is correct for your OS
4. Look at Claude Desktop logs: Help → View Logs
</Callout>

## Setting Up MCP with Cursor

Cursor (the AI-powered code editor) has excellent MCP support with both global and project-specific configuration.

### Step 1: Install Cursor

Download from [cursor.com](https://cursor.com/) and install for your operating system.

### Step 2: Access MCP Settings

Two ways to open MCP configuration in Cursor:

**Method A: Command Palette**
1. Press `Cmd + Shift + P` (macOS) or `Ctrl + Shift + P` (Windows/Linux)
2. Type "MCP Settings"
3. Select "View: Open MCP Settings"
4. Click "Tools & Integrations" → "MCP Tools" → "Add Custom MCP"

**Method B: Direct File Edit**
1. Open `~/.cursor/mcp.json` (global config)
2. Or create `.cursor/mcp.json` in your project root (project-specific)

### Step 3: Add MCP Configuration

Cursor's configuration format is identical to Claude Desktop:

**Global Configuration** (`~/.cursor/mcp.json`):

```json
{
  "mcpServers": {
    "suparank": {
      "command": "npx",
      "args": ["suparank", "my-blog", "sk_live_..."]
    },
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/yourname/projects"
      ]
    }
  }
}
```

**Project-Specific** (`.cursor/mcp.json` in your project):

```json
{
  "mcpServers": {
    "project-docs": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "${workspaceFolder}/docs"
      ]
    }
  }
}
```

<Callout type="info">
**Cursor Limitation**: Cursor supports a maximum of 40 tools simultaneously. If you add multiple MCP servers with many tools, you may need to disable some servers in settings to stay under the limit.
</Callout>

### Step 4: Reload Cursor

After saving the configuration:

1. Press `Cmd + Shift + P` (macOS) or `Ctrl + Shift + P` (Windows)
2. Type "Reload Window"
3. Select "Developer: Reload Window"

Or simply restart Cursor.

### Step 5: Test MCP Tools

In Cursor's AI chat panel:

```
List available MCP tools
```

Then test a content writing workflow:

```
Use Suparank to research keywords for "best espresso machines 2026"
```

## Advanced Configuration: Production Best Practices

### Global vs Local Installation

You have three installation methods for MCP servers:

**Method 1: npx (Temporary, Recommended for Testing)**

```json
{
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-memory"]
}
```

- **Pros**: No installation required, always latest version
- **Cons**: Slower startup (downloads each time), requires internet

**Method 2: Global Installation (Recommended for Production)**

```bash
npm install -g @modelcontextprotocol/server-memory
```

Then configure:

```json
{
  "command": "mcp-server-memory"
}
```

- **Pros**: Faster startup, works offline, version control
- **Cons**: Manual updates required

**Method 3: Project-Specific Installation**

```bash
cd ~/your-project
npm install @modelcontextprotocol/server-memory
```

Configure with full path:

```json
{
  "command": "node",
  "args": ["/Users/yourname/your-project/node_modules/@modelcontextprotocol/server-memory/dist/index.js"]
}
```

- **Pros**: Project isolation, team consistency
- **Cons**: More complex configuration

### Environment Variables & Secrets

Store API keys and credentials in environment variables, not in the config file:

**macOS/Linux** (`~/.zshrc` or `~/.bashrc`):

```bash
export SUPARANK_API_KEY="sk_live_your_key_here"
export NOTION_API_KEY="secret_your_notion_key"
export GITHUB_TOKEN="ghp_your_github_token"
```

Then reference in config:

```json
{
  "mcpServers": {
    "suparank": {
      "command": "npx",
      "args": ["suparank", "project-slug"],
      "env": {
        "SUPARANK_API_KEY": "${SUPARANK_API_KEY}"
      }
    }
  }
}
```

**Windows** (Environment Variables):

1. Search "Environment Variables" in Start Menu
2. Add new user variable: `SUPARANK_API_KEY` = `sk_live_...`
3. Restart Claude Desktop

<Callout type="warning">
**Security**: Never commit API keys to Git repositories. Use environment variables or credential managers. If you accidentally commit a key, revoke it immediately in your dashboard.
</Callout>

### Error Handling & Logging

Enable debug logging for troubleshooting:

**Claude Desktop Debug Mode:**

Add to configuration:

```json
{
  "mcpServers": {
    "suparank": {
      "command": "npx",
      "args": ["suparank", "project", "key"],
      "env": {
        "DEBUG": "mcp:*"
      }
    }
  }
}
```

View logs:
- **macOS**: `~/Library/Logs/Claude/mcp-server-suparank.log`
- **Windows**: `%LOCALAPPDATA%\Claude\logs\`

**Common Errors & Solutions:**

<Table>
| Error | Cause | Solution |
|-------|-------|----------|
| `command not found: npx` | Node.js not installed | Install Node.js from nodejs.org |
| `ENOENT: no such file` | Invalid file path in config | Use absolute paths, check spelling |
| `JSON parse error` | Invalid JSON syntax | Validate at jsonlint.com |
| `Connection refused` | MCP server crashed | Check logs, restart AI client |
| `API key invalid` | Wrong credentials | Verify in dashboard, regenerate if needed |
</Table>

## Troubleshooting Common Issues

### Issue 1: MCP Tools Not Appearing

**Symptoms:** No MCP indicator in AI client after configuration

**Solutions:**

1. **Verify JSON syntax:**
   - Copy config to [jsonlint.com](https://jsonlint.com/)
   - Fix any syntax errors (missing commas, quotes)

2. **Check Node.js installation:**
   ```bash
   node --version  # Should show v18+ or v20+
   npx --version   # Should show version number
   ```

3. **Verify file location:**
   - Claude Desktop: `~/Library/Application Support/Claude/claude_desktop_config.json` (macOS)
   - Cursor: `~/.cursor/mcp.json` (global) or `.cursor/mcp.json` (project)

4. **Restart properly:**
   - Completely quit the application (not just close window)
   - On macOS: `Cmd + Q`
   - Relaunch from Applications folder

5. **Check logs:**
   - Claude Desktop: Help → View Logs
   - Cursor: View → Output → Select "MCP" from dropdown

### Issue 2: "Command not found: npx"

**Cause:** Node.js not installed or not in system PATH

**Solution:**

1. Install Node.js from [nodejs.org](https://nodejs.org/) (LTS version)
2. After installation, restart your terminal
3. Verify: `npx --version`
4. If still not working, add to PATH manually:

**macOS/Linux:**
```bash
echo 'export PATH="/usr/local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

**Windows:**
Search "Environment Variables" → Edit PATH → Add `C:\Program Files\nodejs\`

### Issue 3: MCP Server Crashes Immediately

**Symptoms:** Tools appear briefly then disappear, or error in logs

**Solutions:**

1. **Test the server manually:**
   ```bash
   npx suparank your-project your-api-key
   ```
   Check for error messages

2. **Verify API credentials:**
   - Check API key is correct (not expired/revoked)
   - Verify project slug matches dashboard

3. **Check for port conflicts:**
   - Some MCP servers use specific ports
   - Close other development tools temporarily

4. **Update to latest version:**
   ```bash
   npm uninstall -g suparank
   npm install -g suparank@latest
   ```

### Issue 4: Slow MCP Tool Responses

**Cause:** Using `npx` downloads packages each time

**Solution:** Switch to global installation

```bash
# Install globally
npm install -g suparank

# Update config to use installed version
{
  "command": "suparank",
  "args": ["project-slug", "api-key"]
}
```

### Issue 5: Authentication Errors

**Symptoms:** "Invalid API key" or "Unauthorized" errors

**Solutions:**

1. **Regenerate API key:**
   - Go to your dashboard (app.suparank.io)
   - Settings → API Keys → Regenerate

2. **Check environment variables:**
   ```bash
   echo $SUPARANK_API_KEY  # Should print your key
   ```

3. **Verify key format:**
   - Production: `sk_live_` + 64 characters
   - Test: `sk_test_` + 64 characters

4. **Update config with new key:**
   Edit config file and restart AI client

<Callout type="tip" title="Pro Debugging Tip">
When troubleshooting, start simple:

1. Test with memory server first (minimal dependencies)
2. If that works, issue is with your specific MCP server
3. If that doesn't work, issue is with MCP setup in general
4. Check one variable at a time: Node.js → Config syntax → Server credentials
</Callout>

## Best Practices for MCP Content Automation

### 1. Layer AI + Human Expertise

**Don't:** Publish AI-generated content without review

**Do:** Use MCP to handle research and structure, then add:
- First-hand experience and examples
- Your unique perspective and insights
- Brand voice refinement
- Fact-checking and accuracy verification

### 2. Configure Project-Specific Settings

For Suparank, set up your project config in the dashboard:

- **Brand Voice**: Professional, conversational, technical, etc.
- **Target Audience**: Who you're writing for
- **Primary Keywords**: Your core topics
- **Style Guide**: Specific terminology, formatting preferences

This ensures all MCP-generated content matches your brand.

### 3. Use Workflow Chaining

Combine multiple MCP tools in a single conversation:

```
1. Research keywords for "email marketing automation"
2. Create content brief for top keyword
3. Write full blog post following that brief
4. Generate hero image
5. Run quality check
6. Publish to WordPress as draft
```

The AI executes all steps sequentially, maintaining context throughout.

### 4. Version Control for Content

If using GitHub MCP alongside content tools:

```
1. Write blog post about [topic]
2. Save to filesystem in /blog-drafts
3. Commit to GitHub with message "Draft: [title]"
4. After I review, publish to WordPress
```

This creates an audit trail and enables team collaboration.

### 5. Batch Operations

Use MCP for batch content creation:

```
Create 5 blog post outlines about:
1. Coffee brewing methods
2. Best espresso machines
3. Coffee bean storage
4. Latte art tutorials
5. Coffee grinder buying guide

For each outline, generate:
- Keyword research
- Content brief
- Suggested internal links
```

Process multiple pieces simultaneously.

### 6. Quality Gates

Always run quality checks before publishing:

- Grammar and spelling
- Readability score (aim for grade 8-10)
- SEO requirements (meta description, alt text, etc.)
- Brand consistency
- Fact accuracy

<Callout type="warning">
**E-E-A-T Reminder**: Google's E-E-A-T (Experience, Expertise, Authoritativeness, Trustworthiness) guidelines still apply to AI-generated content. Always add first-hand experience, cite sources, and demonstrate expertise. MCP tools help with efficiency, not replacing human judgment.
</Callout>
